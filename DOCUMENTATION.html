What is Senet?

Senet is one of the oldest known board games, dating back to ancient Egypt (around 3100 BCE). The goal is to move all 7 pieces off the board before your opponent.

getLegalMoves(GameState state, int roll) ‚Üí List<Move>

‚ö†Ô∏è The Most Important Method

Generates all valid moves for the current state and roll.

üìã Algorithm
  1. Get current player's piece positions
  2. FOR each piece position:
    1. Calculate target = position + roll
    2. IF piece on special exit squares (26-30) ‚Üí check special exit conditions
    3. IF target == 26 (House of Happiness) ‚Üí must land exactly
    4. IF target > 26 AND from < 26 ‚Üí ILLEGAL (can't pass 26)
    5. IF target > 30 ‚Üí add EXIT move
    6. IF target is empty ‚Üí add normal MOVE
    7. IF target has opponent ‚Üí add SWAP move
    8. IF target has own piece ‚Üí ILLEGAL (blocked)
  3. Return list of legal moves

applyMove(GameState state, Move move) ‚Üí GameState

üìã Algorithm
  1. Clone current state (immutable pattern)
  2. IF move is EXIT:
    1. Remove piece from board
    2. Increment pieces exited
    3. IF 7 pieces exited ‚Üí game over, set winner
  3. ELSE IF move is SWAP:
    1. Put opponent at fromPosition
    2. Put player at toPosition

  5. Return new state

üß† Expectiminimax Algorithm

Why Expectiminimax?

In games with randomness (dice, cards), we cannot use standard Minimax because:

  • We don't know what dice will roll
  • Different rolls lead to different legal moves
  • We must consider all possible outcomes

Solution: Insert CHANCE nodes that calculate expected values.

Algorithm Structure

Move getBestMove(GameState state, int roll, int maxDepth)

Entry point for the AI. Finds the best move for given state and roll.

üìã Algorithm
  1. Reset statistics
  2. Get all legal moves for current roll
  3. IF no legal moves ‚Üí return null
  4. IF only one move ‚Üí return that move (no search needed)
  5. bestValue = -‚àû, bestMove = null
  6. FOR each legal move:
    1. Apply move to get next state
    2. Switch player (opponent's turn)
    3. value = chanceNode(nextState, depth-1, -‚àû, +‚àû, false)
    4. IF value > bestValue ‚Üí update bestValue, bestMove
  7. Return bestMove
double chanceNode(state, depth, alpha, beta, isMaxPlayer)

Calculates expected value over all possible dice outcomes.

üìã Algorithm
  1. Increment CHANCE node counter
  2. IF depth == 0 OR game over ‚Üí return Heuristic.evaluate()
  3. expectedValue = 0.0
  4. FOR each roll in [1, 2, 3, 4, 5]:
    1. probability = getProbability(roll)
    2. IF isMaxPlayer ‚Üí value = maxNode(...)
    3. ELSE ‚Üí value = minNode(...)
    4. expectedValue += probability √ó value
  5. Return expectedValue
üìê Mathematical Formula
E[V] = Œ£ P(roll) √ó Value(roll)
     = 0.25√óV(1) + 0.375√óV(2) + 0.25√óV(3) + 0.0625√óV(4) + 0.0625√óV(5)
double maxNode(state, depth, alpha, beta, roll)

Computer's turn - tries to MAXIMIZE the score.

üìã Algorithm
  1. Get legal moves for given roll
  2. IF no legal moves ‚Üí clone state, switch player, return chanceNode()
  3. maxValue = -‚àû
  4. FOR each legal move:
    1. Apply move, switch player
    2. value = chanceNode(nextState, depth-1, alpha, beta, false)
    3. maxValue = max(maxValue, value)
    4. alpha = max(alpha, value)
    5. IF beta ‚â§ alpha ‚Üí BREAK (Alpha-Beta Pruning)
  5. Return maxValue
double minNode(state, depth, alpha, beta, roll)

Opponent's turn - tries to MINIMIZE the score.

üìã Algorithm
  1. Get legal moves for given roll
  2. IF no legal moves ‚Üí clone state, switch player, return chanceNode()
  3. minValue = +‚àû
  4. FOR each legal move:
    1. Apply move, switch player
    2. value = chanceNode(nextState, depth-1, alpha, beta, true)
    3. minValue = min(minValue, value)
    4. beta = min(beta, value)
    5. IF beta ‚â§ alpha ‚Üí BREAK (Alpha-Beta Pruning)
  5. Return minValue

Alpha-Beta Pruning

Alpha-Beta pruning optimizes the search by eliminating branches that cannot affect the final decision.

Œ± (Alpha)

Best value MAX can guarantee (starts at -‚àû)

Œ≤ (Beta)

Best value MIN can guarantee (starts at +‚àû)

‚ö° Pruning Condition: Œ≤ ‚â§ Œ±

When this condition is met, we can skip remaining branches because they won't affect the result.

Evaluation Formula

Score = (PiecesExited √ó 1000) + (Advancement √ó 10) + (Safety √ó 5)
      + (SpecialSquares √ó 20) + (Blocking √ó 15)

Scoring Components

1. Pieces Exited (Weight: 1000)

Most important - directly related to winning condition.

score += (myExited - opponentExited) √ó 1000

2. Advancement Score (Weight: 10)

Rewards pieces closer to exiting. Uses quadratic formula.

for (pos : positions)
    playerSum += pos √ó pos; // Quadratic

3. Safety Score (Weight: 5)

Rewards pieces in safe zones.

if (pos >= 26) safetyScore += 3.0;
else if (pos > 15) safetyScore += 1.0;

4. Special Squares (Weight: 20)

Evaluates special square occupancy.

Position 27 (Water)-50
Position 26 (Happiness)+25
Position 28 (Three Truths)+30
Position 29 (Re-Atoum)+35
Position 30 (Horus)+40
